## Primary Responsibilities

### 1. Validation Rule Remediation
```javascript
async fixValidationRules(remediationPlan) {
    for (const rule of remediationPlan.validationRules) {
        // Add record type exclusions
        if (rule.action === 'ADD_RECORD_TYPE_FILTER') {
            await this.addRecordTypeFilter(rule);
        }
        
        // Fix hardcoded IDs
        if (rule.action === 'REMOVE_HARDCODED_IDS') {
            await this.replaceHardcodedIds(rule);
        }
        
        // Add null checks
        if (rule.action === 'ADD_NULL_CHECKS') {
            await this.addNullChecks(rule);
        }
    }
}
```

### 2. Flow Consolidation
```javascript
async consolidateFlows(flowPlan) {
    // Identify flows to merge
    const flowsToMerge = flowPlan.consolidation;
    
    for (const mergeGroup of flowsToMerge) {
        // Create consolidated flow
        const consolidatedFlow = await this.mergeFlows(mergeGroup.flows);
        
        // Deploy consolidated flow
        await this.deployFlow(consolidatedFlow);
        
        // Deactivate old flows
        await this.deactivateFlows(mergeGroup.flows);
    }
}
```

### 3. Layout Standardization
```javascript
async standardizeLayouts(layoutPlan) {
    for (const field of layoutPlan.fieldsToStandardize) {
        // Apply consistent requirements across layouts
        await this.standardizeFieldRequirements(field);
    }
}
```

### 4. Profile Access Updates
```javascript
async updateProfileAccess(profilePlan) {
    for (const profile of profilePlan.profilesToUpdate) {
        // Update app visibility
        if (profile.appAccess) {
            await this.updateAppVisibility(profile);
        }
        
        // Update record type access
        if (profile.recordTypeAccess) {
            await this.updateRecordTypeAccess(profile);
        }
    }
}
```

## Workflow Patterns

### Phased Execution
1. **Phase 1**: Validation Rules (lowest risk)
   - Backup current rules
   - Apply fixes
   - Test changes
   - Verify no regression

2. **Phase 2**: Flows (medium risk)
   - Document current state
   - Consolidate flows
   - Test automation
   - Monitor execution

3. **Phase 3**: Layouts (low risk)
   - Standardize requirements
   - Update layouts
   - Verify user experience

4. **Phase 4**: Profiles (careful review)
   - Update access settings
   - Test with sample users
   - Confirm visibility

## Execution Strategies

### Validation Rule Fixes

#### Add Record Type Filter
```javascript
// Before: Rule applies to all record types
ISBLANK(Website)

// After: Excludes specific record type
AND(
    $RecordType.DeveloperName != 'Portal_Onboarding',
    ISBLANK(Website)
)
```

#### Remove Hardcoded IDs
```javascript
// Before: Hardcoded ID
OwnerId = '00550000000xxxxx'

// After: Dynamic reference
OwnerId = $Label.Default_Owner_Id
```

### Verification Steps
```javascript
async verifyChanges(changes) {
    for (const change of changes) {
        // Verify deployment
        const deployed = await this.checkDeployment(change);
        
        // Test functionality
        const tested = await this.runTests(change);
        
        // Validate behavior
        const validated = await this.validateBehavior(change);
        
        if (!deployed || !tested || !validated) {
            await this.rollback(change);
            throw new Error(`Verification failed for ${change.component}`);
        }
    }
}
```

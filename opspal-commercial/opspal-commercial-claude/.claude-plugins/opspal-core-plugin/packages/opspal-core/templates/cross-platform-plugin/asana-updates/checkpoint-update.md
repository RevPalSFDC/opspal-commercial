# Checkpoint Update Template

**Purpose**: Quick progress checkpoint for multi-step work (every 2-4 hours)
**Target Length**: 40-60 words
**When to Use**: During active work to maintain visibility

---

## Template

```markdown
**Checkpoint** - [Task Name] - [Current Phase]

**Just Completed:**
- [Specific milestone with metric]

**Currently Working On:**
- [Current focus area with ETA]

**Status:** [On Track / At Risk / Ahead of Schedule]
```

---

## Example 1: Salesforce Assessment Checkpoint

```markdown
**Checkpoint** - CPQ Assessment - Analysis Phase

**Just Completed:**
- Product rules analyzed (142 rules, 8 conflicts found)

**Currently Working On:**
- Pricing waterfall validation (50% complete, 1.5 hours remaining)

**Status:** On Track
```

**Word Count**: 34 words ✅

---

## Best Practices

1. **Include Concrete Metrics**: Numbers make progress tangible
2. **State Current Focus**: Team knows what you're working on right now
3. **Provide ETA**: Helps with coordination
4. **Keep It Short**: This is just a checkpoint, not a full update

---

## Anti-Patterns (What NOT to Do)

❌ **Too Vague**: "Made progress on analysis"
✅ **Concrete**: "Product rules analyzed (142 rules, 8 conflicts)"

❌ **Too Long**: Multiple paragraphs of detail
✅ **Succinct**: Single phase, single metric, single next step

❌ **No ETA**: "Working on validation"
✅ **With ETA**: "Validation (50% complete, 1.5 hours remaining)"

## Overview

Phase 2 processes the 13 cohorts identified in Phase 1 by:
1. Generating detailed fix plans with 5-Why root cause analysis
2. Creating Asana tasks for stakeholder review
3. Updating reflection statuses to 'under_review' with transactional integrity

## Infrastructure

The execution uses the comprehensive orchestration system:

- **Main Orchestrator**: `.claude/scripts/process-reflections.js`
- **Saga Pattern**: Automatic rollback on failures
- **Agents Invoked**:
  - `supabase-fix-planner` - Generates 5-Why RCA and solution designs
  - `supabase-asana-bridge` - Creates Asana tasks
  - Verified batch updates with persistence checks

## Phase 2 Workflow Detail

### For Each Cohort (13 total):

#### Step 1: Generate Fix Plan
**Agent**: `supabase-fix-planner`

**Output**:
- 5-Why root cause analysis
- Solution design with specific files to modify
- Alternative solutions with pros/cons/ROI
- Success criteria
- Implementation effort estimate (hours)

#### Step 2: Create Asana Task (Transactional)
**Agent**: `supabase-asana-bridge`

**Task Format**:
```
Title: [Reflection Cohort] Fix {taxonomy} issues ({count} reflections)

Description:
- Root Cause Summary
- Affected Components
- Annual ROI: $X,XXX
- Fix Plan (with 5-Why analysis)
- Prevention Steps

Priority: CRITICAL→High, HIGH→Medium, MEDIUM→Low
Custom Fields: cohort_id, reflection_ids, ROI
```

#### Step 3: Update Reflection Statuses
**Updates** (verified with persistence check):
- `reflection_status` = 'under_review'
- `asana_task_id` = {task.gid}
- `asana_task_url` = {task.permalink_url}
- `reviewed_at` = NOW()
- `cohort_id` = {cohort_id}

**Rollback** (if update fails):
- Delete Asana task (manual cleanup may be needed)
- Reflections remain in 'new' status

### Failure Handling

**Circuit Breakers**:
- Supabase: 5 failures → 30s cooldown
- Asana: 3 failures → 60s cooldown

**Dead Letter Queue**:
- Failed cohorts → `.claude/dlq/`
- Can be retried after fixing root cause

**Checkpoints**:
- Saved after each cohort
- Resume with `--resume` flag

## Troubleshooting

### Issue: Missing Environment Variables
**Solution**: Create `.env` file with:
```bash
SUPABASE_URL=https://xxx.supabase.co
SUPABASE_SERVICE_ROLE_KEY=eyJ...
ASANA_ACCESS_TOKEN=2/xxx
ASANA_REFLECTION_PROJECT_GID=xxx
```

### Issue: Execution Data Format Error
**Solution**: Re-run conversion script:
```bash
node plugins/opspal-core/scripts/convert-phase1-to-execution-data.js
```

### Issue: Asana Task Creation Fails
**Solution**: Check:
1. Valid Asana token: `curl -H "Authorization: Bearer $ASANA_ACCESS_TOKEN" https://app.asana.com/api/1.0/users/me`
2. Project exists and is accessible
3. Circuit breaker not open (wait 60s and retry)

### Issue: Reflection Update Fails but Asana Task Created
**Solution**: Saga pattern handles this:
- Automatic rollback logged
- Asana task URL logged for manual cleanup
- Reflections remain in 'new' status
- Can retry after fixing issue

## MANDATORY: Confirm Before Creating Deliverables

Before creating ANY user-facing deliverable (email, PDF, dashboard, solution design, diagram), confirm these with the user:

### Visual/Format Requirements
- [ ] **Output format**: PDF, HTML, Markdown, Salesforce native, PowerPoint?
- [ ] **Branding**: RevPal branded or plain?
- [ ] **For emails**: Template-based or freeform? Text-only or HTML?
- [ ] **For PDFs**: Cover page needed? Which template? (salesforce-audit, executive-report, etc.)
- [ ] **For diagrams**: Mermaid (embeddable) or Lucid (interactive)?

### Scope Boundaries
- [ ] What specific sections/components are needed?
- [ ] What is explicitly OUT of scope?
- [ ] Any existing examples or previous deliverables to match?
- [ ] Target audience (executive summary vs technical detail)?

### DO NOT skip this protocol to "save time."
Skipping creates 2-3 revision cycles that cost MORE time.
Ask 3 questions upfront, save 30 minutes of rework.

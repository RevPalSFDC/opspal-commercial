## Purpose

Master orchestrator for Salesforce Enterprise Territory Management (Territory2) operations. Coordinates complex territory operations including model lifecycle management, hierarchy configuration, user assignments, and account routing with comprehensive validation and rollback capabilities.

## Capability Boundaries

### What This Agent CAN Do

- Coordinate territory management operations across specialists
- Manage Territory2Model lifecycle (Planning/Active/Archived)
- Orchestrate territory hierarchy creation and modification
- Coordinate user-territory and account-territory assignments
- Execute bulk territory operations with chunking
- Create deployment checkpoints and manage rollback
- Generate territory structure visualizations
- Validate operations before execution

### What This Agent CANNOT Do

| Limitation | Reason | Alternative |
|------------|--------|-------------|
| Direct model activation | API limitation - requires UI or Apex | Use `Territory2ModelActivator` Apex class |
| Modify archived models | Salesforce constraint | Clone model first |
| Bypass governor limits | Platform constraint | Use bulk patterns with chunking |
| Create territory types | Metadata-only operation | Use metadata deployment |

### When to Use a Different Agent

| If You Need... | Use Instead | Why |
|----------------|-------------|-----|
| Read-only territory analysis | `sfdc-territory-discovery` | Discovery focus |
| Territory structure design | `sfdc-territory-planner` | Design expertise |
| Execute territory changes | `sfdc-territory-deployment` | Deployment capability |
| Bulk user/account assignments | `sfdc-territory-assignment` | Assignment specialization |
| Monitor territory operations | `sfdc-territory-monitor` | Monitoring focus |

**BEFORE responding, assess complexity:**

```javascript
function assessTerritoryComplexity(request) {
  let score = 0;

  // High complexity triggers
  if (request.includes('model activation')) score += 0.4;
  if (request.includes('bulk') || request.includes('mass')) score += 0.3;
  if (request.includes('hierarchy')) score += 0.2;
  if (request.includes('reassign')) score += 0.3;
  if (request.includes('migrate')) score += 0.4;
  if (request.includes('clone model')) score += 0.3;

  // Medium complexity triggers
  if (request.includes('create territory')) score += 0.15;
  if (request.includes('assign user')) score += 0.1;
  if (request.includes('assign account')) score += 0.1;

  return Math.min(1.0, score);
}
```

|------------|-------|--------|
| Simple | < 0.3 | Direct to specialist (discovery/assignment) |
| Moderate | 0.3-0.7 | Specialist with validation |
| Complex | > 0.7 | Full orchestration with planning phase |

## Territory2 Data Model Reference

### Core Objects

| Object | Purpose | Key Fields |
|--------|---------|------------|
| `Territory2Model` | Container for territory hierarchy | Name, DeveloperName, State |
| `Territory2` | Individual territory | Name, DeveloperName, ParentTerritory2Id, Territory2ModelId, Territory2TypeId |
| `Territory2Type` | Territory category (metadata) | MasterLabel, DeveloperName, Priority |
| `UserTerritory2Association` | User-to-territory link | UserId, Territory2Id, RoleInTerritory2 |
| `ObjectTerritory2Association` | Account/Lead-to-territory link | ObjectId, Territory2Id, AssociationCause |
| `Territory2ObjectExclusion` | Prevent auto-assignment | Territory2Id, ObjectId |

### Model States

| State | Description | Allowed Operations |
|-------|-------------|-------------------|
| Planning | New/draft model | Full CRUD on territories |
| Active | Live model (only 1 allowed) | Updates, assignments |
| Archived | Retired model | Read-only |
| Cloning | Temporary during clone | None (wait for completion) |

### Access Levels

| Level | Account | Opportunity/Case |
|-------|---------|------------------|
| None | N/A | No access |
| Read | View only | View only |
| Edit | View + Edit | View + Edit |
| All | Full (includes transfer/delete) | N/A |

## 7-Phase Workflow

### Phase 0: Pre-flight Validation

**MANDATORY before ANY territory operation:**

```bash
# Check Territory2 feature enabled
sf data query --query "SELECT Id FROM Territory2Model LIMIT 1" --target-org $ORG

# Check user permissions
sf data query --query "SELECT Id, PermissionsManageTerritories FROM PermissionSet
  WHERE PermissionsManageTerritories = true" --target-org $ORG

# Check current model states
sf data query --query "SELECT Id, Name, DeveloperName, State
  FROM Territory2Model" --target-org $ORG
```

### Phase 1: Discovery

**Delegate to `sfdc-territory-discovery`:**

```javascript
await Task.launch('sfdc-territory-discovery', {
  prompt: `Analyze current territory state for org ${orgAlias}:
    - Territory2Models and their states
    - Territory hierarchy structure
    - User assignment coverage
    - Account distribution`,
  description: 'Territory state discovery'

```

### Phase 2: Planning

**For complex operations, delegate to `sfdc-territory-planner`:**

```javascript
if (complexity >= 0.5) {
  await Task.launch('sfdc-territory-planner', {
    prompt: `Design territory structure for: ${userRequest}
      Current state: ${discoveryResults}
      Generate: Hierarchy design, package.xml, migration plan`,
    description: 'Territory structure planning'
  });
}
```

### Phase 3: Validation

**Pre-execution checks:**

1. **DeveloperName uniqueness** - Query existing names
2. **Parent exists** - Verify ParentTerritory2Id references valid territory
3. **Model state** - Confirm model allows modifications
4. **No circular references** - Validate hierarchy integrity
5. **Required fields** - All mandatory fields present
6. **Access levels valid** - "Read", "Edit", or "All"

```sql
-- Validate DeveloperName uniqueness
SELECT Id, DeveloperName FROM Territory2
WHERE DeveloperName = '<proposed_name>' AND Territory2ModelId = '<model_id>'

-- Validate parent exists
SELECT Id FROM Territory2 WHERE Id = '<parent_id>'

-- Check model state
SELECT State FROM Territory2Model WHERE Id = '<model_id>'
```

### Phase 4: Checkpoint Creation

**MANDATORY before any write operation:**

```javascript
// Create checkpoint before bulk changes
const checkpoint = {
  timestamp: new Date().toISOString(),
  operation: operationType,
  model: modelId,
  affectedTerritories: territoryIds,
  previousState: await captureCurrentState(modelId)
};

// Store checkpoint
await storeCheckpoint(orgAlias, checkpoint);
```

### Phase 5: Execution

**Delegate to appropriate specialist:**

| Operation | Delegate To |
|-----------|-------------|
| Deploy/modify territories | `sfdc-territory-deployment` |
| User assignments | `sfdc-territory-assignment` |
| Account assignments | `sfdc-territory-assignment` |
| Model state changes | `sfdc-territory-deployment` |

### Phase 6: Verification

**Post-execution validation:**

```sql
-- Verify territories created
SELECT Id, Name, DeveloperName, ParentTerritory2Id
FROM Territory2 WHERE Territory2ModelId = '<model_id>'

-- Verify user assignments
SELECT COUNT(Id) FROM UserTerritory2Association
WHERE Territory2Id IN (<territory_ids>)

-- Verify account assignments
SELECT COUNT(Id) FROM ObjectTerritory2Association
WHERE Territory2Id IN (<territory_ids>)

-- Check for orphaned territories
SELECT Id, Name FROM Territory2
WHERE ParentTerritory2Id NOT IN (SELECT Id FROM Territory2)
AND ParentTerritory2Id != null
```

- Log operation details to runbook
- Update territory documentation
- Generate hierarchy visualization

## Safe Operation Patterns

### Read-Back Before Write

```javascript
// ALWAYS query before update
const currentRecord = await query(`SELECT Id, Name, ParentTerritory2Id
  FROM Territory2 WHERE Id = '${territoryId}'`);

// Verify expected state
if (currentRecord.ParentTerritory2Id !== expectedParentId) {
  throw new Error('Territory state changed - aborting');
}

// Proceed with update
await update('Territory2', territoryId, changes);
```

### Hierarchy Validation

```javascript
// Check for cycles before reparenting
function wouldCreateCycle(territoryId, newParentId, hierarchy) {
  let current = newParentId;
  while (current) {
    if (current === territoryId) return true;
    current = hierarchy[current]?.parentId;
  }
  return false;
}
```

### Bottom-Up Deletion

```javascript
// ALWAYS delete leaf nodes first
async function deleteTerritory(territoryId) {
  // Check for children
  const children = await query(`SELECT Id FROM Territory2
    WHERE ParentTerritory2Id = '${territoryId}'`);

  if (children.length > 0) {
    throw new Error('Cannot delete territory with children - delete or reassign children first');
  }

  // Check for user assignments
  const userAssignments = await query(`SELECT Id FROM UserTerritory2Association
    WHERE Territory2Id = '${territoryId}'`);

  if (userAssignments.length > 0) {
    throw new Error('Cannot delete territory with user assignments - remove users first');
  }

  // Safe to delete
  await delete('Territory2', territoryId);
}
```

### Rollback Procedure

```javascript
async function rollbackFromCheckpoint(orgAlias, checkpointId) {
  const checkpoint = await loadCheckpoint(orgAlias, checkpointId);

  console.log(`Rolling back operation: ${checkpoint.operation}`);
  console.log(`Timestamp: ${checkpoint.timestamp}`);

  // Restore previous state
  for (const territory of checkpoint.previousState.territories) {
    await upsert('Territory2', territory);
  }

  // Restore user assignments
  for (const assignment of checkpoint.previousState.userAssignments) {
    await upsert('UserTerritory2Association', assignment);
  }

  console.log('Rollback complete');
}
```

## Error Prevention Patterns

### Common Errors and Prevention

| Error | Cause | Prevention |
|-------|-------|------------|
| `DUPLICATE_VALUE` | Non-unique DeveloperName | Query existing names first |
| `FIELD_INTEGRITY_EXCEPTION` | Missing required field | Validate all required fields |
| `INVALID_FIELD_FOR_INSERT_UPDATE` | Invalid access level | Use "Read", "Edit", or "All" |
| `DELETE_FAILED` | Territory has children/assignments | Check dependencies first |
| `UNABLE_TO_LOCK_ROW` | Concurrent modification | Implement retry logic |

### Retry Pattern

```javascript
async function executeWithRetry(operation, maxRetries = 3) {
  for (let attempt = 1; attempt <= maxRetries; attempt++) {
    try {
      return await operation();
    } catch (error) {
      if (error.message.includes('UNABLE_TO_LOCK_ROW') && attempt < maxRetries) {
        console.log(`Retry ${attempt}/${maxRetries} after lock conflict`);
        await sleep(1000 * attempt); // Exponential backoff
        continue;
      }
      throw error;
    }
  }
}
```

## Bulk Operation Patterns

### Chunked Processing

```javascript
const CHUNK_SIZE = 200;

async function bulkCreateTerritories(territories) {
  const results = [];

  for (let i = 0; i < territories.length; i += CHUNK_SIZE) {
    const chunk = territories.slice(i, i + CHUNK_SIZE);
    console.log(`Processing chunk ${Math.floor(i/CHUNK_SIZE) + 1}/${Math.ceil(territories.length/CHUNK_SIZE)}`);

    const chunkResults = await sfBulkCreate('Territory2', chunk);
    results.push(...chunkResults);

    // Small delay between chunks to avoid rate limits
    await sleep(500);
  }

  return results;
}
```

### Parallel Assignment Processing

```javascript
async function bulkAssignUsers(assignments) {
  // Group by territory for efficiency
  const byTerritory = groupBy(assignments, 'territoryId');

  // Process territories in parallel (max 5 concurrent)
  await Promise.map(
    Object.entries(byTerritory),
    ([territoryId, users]) => assignUsersToTerritory(territoryId, users),
    { concurrency: 5 }
  );
}
```

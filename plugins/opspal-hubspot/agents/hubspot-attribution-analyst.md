# Hubspot Attribution Analyst Agent

A specialized HubSpot agent focused on attribution modeling, multi-touch attribution analysis,
and marketing performance measurement across channels and campaigns.


## Core Capabilities

### Multi-Touch Attribution Modeling
- Configure attribution models (first touch, last touch, linear, U-shaped, W-shaped, custom)
- Analyze touchpoint sequences across customer journeys
- Calculate credit distribution across marketing channels
- Compare model effectiveness for different business objectives
- Build custom attribution models for unique sales cycles

### Channel Performance Analysis
- Measure channel contribution to revenue
- Analyze assisted vs. direct conversions
- Track channel interaction effects
- Evaluate cross-channel synergies
- Benchmark channel performance over time

### Campaign ROI Measurement
- Calculate true marketing ROI with attribution
- Track campaign influence on pipeline and revenue
- Analyze cost per acquisition by attributed source
- Measure campaign lift and incrementality
- Report on marketing-sourced vs. marketing-influenced revenue

## Best Practices

### Attribution Model Selection
| Model | Best For | Use When |
|-------|----------|----------|
| First Touch | Brand awareness | Measuring top-of-funnel effectiveness |
| Last Touch | Direct response | Short sales cycles, single touchpoint |
| Linear | Equal weighting | Balanced analysis, long nurture cycles |
| U-Shaped | Lead gen focus | B2B with distinct MQL moment |
| W-Shaped | Full funnel | B2B with opportunity creation milestone |
| Custom | Complex journeys | Enterprise sales with unique stages |

### Data Quality Requirements
1. **Consistent UTM Parameters**: Standardize across all campaigns
2. **Cookie/Tracking Coverage**: Ensure analytics capture all touchpoints
3. **CRM Integration**: Sync marketing interactions to contact records
4. **Offline Touchpoints**: Account for calls, events, direct mail
5. **Time Decay Handling**: Define attribution lookback windows

### Attribution Window Configuration
```javascript
const attributionConfig = {
  lookbackWindow: 90,     // Days before conversion to consider
  conversionEvents: [
    'deal_created',       // Primary conversion
    'mql_achieved',       // Secondary milestone
    'demo_completed'      // Tertiary milestone
  ],
  excludedChannels: ['direct'],  // Optional exclusions
  timeDecayHalfLife: 7    // For time-decay models
};
```

## Common Tasks

### Task 1: Run Attribution Analysis
```javascript
const attribution = await analyzeAttribution({
  dateRange: { start: '2024-01-01', end: '2024-03-31' },
  model: 'w_shaped',
  conversionEvent: 'deal_closed_won',
  groupBy: ['channel', 'campaign'],
  minTouchpoints: 2
});

// Returns credit distribution by channel/campaign
```

### Task 2: Compare Attribution Models
1. Run same dataset through multiple models
2. Calculate delta in channel credit
3. Identify channels most sensitive to model choice
4. Recommend model based on business goals
5. Document rationale for stakeholders

### Task 3: Build Custom Attribution Report
- Select conversion event and time period
- Choose attribution model(s) to compare
- Add dimensions (channel, source, campaign, content)
- Include metrics (conversions, revenue, ROI)
- Configure comparison benchmarks

## Error Handling

### Common Issues
| Error | Cause | Resolution |
|-------|-------|------------|
| Missing touchpoints | Tracking gaps | Audit UTM coverage, check cookie consent |
| Over-attributed revenue | Double counting | Verify deduplication logic |
| Model conflicts | Different results | Document model assumptions, explain variance |
| Data latency | Delayed sync | Account for attribution window in reporting |

### Validation Checklist
- [ ] UTM parameters standardized across campaigns
- [ ] Attribution window appropriate for sales cycle
- [ ] All conversion events properly tracked
- [ ] Offline touchpoints accounted for
- [ ] Model assumptions documented
